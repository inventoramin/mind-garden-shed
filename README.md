# میز آرشیو — Knowledge Management News

یک نمونه اپلیکیشن مدیریت دانش با پشتیبانی از زبان فارسی و چیدمان راست‌به‌چپ. صفحه اصلی فهرست خبرها را نمایش می‌دهد و هر خبر با کلیک روی عنوان، در صفحه اختصاصی خود باز می‌شود.

## پیش‌نیازها

- Node.js 20 یا جدیدتر
- npm

## اجرای پروژه در محیط توسعه

```bash
npm install
npm run dev
```

پس از اجرا، آدرس محلی نمایش‌داده‌شده در ترمینال را در مرورگر باز کنید.

## تنظیم آدرس API خارجی

برای دریافت خبرها از بک‌اند خارجی، متغیر زیر را در محیط اجرای پروژه تنظیم کنید:

```bash
VITE_NEWS_API_BASE_URL=https://your-backend.com/api npm run dev
```

اگر این متغیر تنظیم نشود، اپ از داده‌های mock فارسی داخل `src/data/news.ts` استفاده می‌کند.

## اسکریپت‌های کاربردی

```bash
npm run dev       # اجرای محیط توسعه

```

## ساختار اصلی پیاده‌سازی

- `src/routes/index.tsx`: صفحه فهرست خبرها
- `src/routes/news.$slug.tsx`: مسیر پویا برای نمایش متن کامل هر خبر
- `src/data/news.ts`: داده mock فارسی خبرها
- `src/styles.css`: توکن‌های طراحی، فونت‌ها، رنگ‌ها و تنظیمات پایه RTL
- `src/routes/__root.tsx`: پوسته اصلی HTML، متادیتا، `lang="fa"` و `dir="rtl"`

## پشتیبانی فارسی و RTL

برای پشتیبانی راست‌به‌چپ:

1. در `src/routes/__root.tsx` مقدارهای زیر روی تگ HTML تنظیم شده‌اند:
   ```tsx
   <html lang="fa" dir="rtl">
   ```
2. متن‌های نمونه در `src/data/news.ts` فارسی هستند.
3. چیدمان و فلش‌های ناوبری برای خواندن راست‌به‌چپ تنظیم شده‌اند.
4. فونت‌های مناسب فارسی در `src/styles.css` به عنوان فونت‌های اصلی تعریف شده‌اند.

## اتصال به بک‌اند

اپ فعلی برای بک‌اند خارجی این endpointها را می‌خواند:

```http
GET /news
GET /news/:slug
```

آدرس کامل با مقدار `VITE_NEWS_API_BASE_URL` ساخته می‌شود؛ برای مثال اگر مقدار آن `https://your-backend.com/api` باشد، اپ از `https://your-backend.com/api/news` و `https://your-backend.com/api/news/:slug` استفاده می‌کند.

پاسخ `GET /news` می‌تواند یکی از این دو شکل باشد:

```json
[
  {
    "slug": "sample-news",
    "title": "عنوان خبر",
    "category": "دسته‌بندی",
    "date": "۲ اردیبهشت ۱۴۰۵",
    "readTime": "۴ دقیقه مطالعه",
    "summary": "خلاصه خبر",
    "body": ["پاراگراف اول", "پاراگراف دوم"]
  }
]
```

یا:

```json
{ "articles": [] }
```

پاسخ `GET /news/:slug` می‌تواند خود آبجکت خبر یا این شکل باشد:

```json
{ "article": { "slug": "sample-news", "title": "عنوان خبر", "category": "دسته‌بندی", "date": "۲ اردیبهشت ۱۴۰۵", "readTime": "۴ دقیقه مطالعه", "summary": "خلاصه خبر", "body": ["متن خبر"] } }
```

برای اتصال این اپ به بک‌اند داخلی، Lovable Cloud را فعال کنید تا دیتابیس، احراز هویت، ذخیره‌سازی فایل و توابع سمت سرور در دسترس باشد.

پیشنهاد پیاده‌سازی:

1. یک جدول برای خبرها بسازید، مثلاً `news_articles` با ستون‌های زیر:
   - `id`
   - `slug`
   - `title`
   - `category`
   - `published_at`
   - `read_time`
   - `summary`
   - `body`
2. داده‌های mock داخل `src/data/news.ts` را با درخواست به بک‌اند جایگزین کنید.
3. در loader مسیر `src/routes/news.$slug.tsx`، خبر را بر اساس `slug` از دیتابیس بخوانید.
4. برای صفحه اصلی، فهرست خبرها را از دیتابیس دریافت کنید و فقط اطلاعات خلاصه را نمایش دهید.
5. اگر پنل مدیریت اضافه می‌کنید، نقش‌ها را در جدول جداگانه نگه دارید و دسترسی‌ها را سمت سرور اعتبارسنجی کنید.

نمونه مسیر فعلی برای خبرها:

```tsx
<Link to="/news/$slug" params={{ slug: article.slug }}>
  {article.title}
</Link>
```

در حالت اتصال به بک‌اند، همین `slug` کلید اصلی برای پیدا کردن خبر در دیتابیس خواهد بود.
